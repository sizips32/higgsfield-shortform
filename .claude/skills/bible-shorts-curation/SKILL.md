# bible-shorts-curation

## 목적
렌더 후보 또는 제작 패키지를 유튜브 쇼츠 업로드 직전 상태로 정리한다. AI 점수는 참고하고, 최종 선택은 사용자의 판단을 남긴다.

## 워크플로우
1. `clips.json`, `higgsfield-plan.md`, `shotlist.md` 중 가능한 파일을 읽는다.
2. 후보별로 훅, 명확성, 신학 안전성, 캐릭터 일관성, 모바일 가독성, 리텐션을 평가한다.
3. Virality Predictor 결과가 있으면 참고 점수로만 병기한다.
4. 사용자에게 최종 컷 선택을 요청하거나, `auto` 모드에서는 추천 순서를 제시하고 `사용자 확인 필요`로 표시한다.
5. 제목/설명/해시태그/고정 댓글을 만든다.

## 산출물
`projects/<slug>/briefs/selects.md`:
- 후보별 평가.
- 추천 편집 순서.
- 사용자 선택 또는 자동 추천 표시.
- 재렌더 필요 컷.

`projects/<slug>/briefs/youtube-package.md`:
- 제목 5안.
- 설명 1안.
- 해시태그 8~12개.
- 고정 댓글 2안.
- 자막/사운드/색보정 체크리스트.

## 검증 체크
- 제목이 성경 내용을 왜곡하지 않는가?
- 미성년자에게 과한 공포나 정죄를 유발하지 않는가?
- 첫 2초 훅이 업로드 패키지에도 살아 있는가?
- 구절 출처가 설명 또는 자막에 남아 있는가?
